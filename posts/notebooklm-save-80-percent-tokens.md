# 用好 NotebookLM 立省 80% Token

上个月作者把 Claude 从 Pro 升到 Max，$200 一个月，心想这下总该用够用了吧。结果第五天，本周额度就耗尽了。

翻使用日志才看清楚钱花在哪。一下午调研 47 篇论文，单次会话就吃掉一周 10% 的额度。问题不在 Claude 不够强，而在于一直让它干它不擅长的事，当全文检索引擎。

一坨 50k 字符的日志塞进对话，问一个问题，日志全文就要被算一次 input token；再问一个，即使命中了 prompt cache，主会话也会随着轮次缓慢累加。更糟的是 cache 有 1h TTL，间隔久了就得重写一次全价。

所以真正省 Claude token 的办法，不是开 cache，而是让重数据一开始就不进 Claude。

## 一句话论点

让 NotebookLM 存储语料和检索，Claude 只做推理和编排。NotebookLM 是老师，Claude 是助手，你是课题负责人。

## 1. NotebookLM 是什么

NotebookLM 有几个很强的优点：

- 免费档 50 个源，Pro 档 300 个源
- 上传、索引、生成、对话都由 Google 算力承担
- 除了问答，还能自动生成音频播客、思维导图、PPT、闪卡等
- 支持 PDF、网页 URL、YouTube 字幕、Google Docs、纯文本、图片 OCR、音频转写等多种格式

它特别适合一件事，坐下来问几个问题。

第一次接触 NotebookLM，是作者朋友用它处理论文 reading list。以前在 PDF 里逐篇 Ctrl-F，现在把全部论文丢进一个 notebook，问“谁支持 X 观点、谁反对、分歧集中在哪几个变量”，答案会直接带引用甩出来。一周能省十几个小时。

## 2. 为什么要再套一层 Claude

NotebookLM 虽然强，但卡在两个地方：

- 调研一个课题时，要在网页、原文、搜索、下载之间疯狂切 tab，心流被切烂
- 跟本地工具是两个世界，不能帮你跑命令、grep、起 pod、做本地实验

NotebookLM 网页把自己定位成终点，而作者想让它成为流水线上的一环。于是 Claude 登场：让 Claude 负责调度、写代码、整理结果，NotebookLM 负责答领域知识。

## 3. 装 skill，10 分钟动手

基础安装：

```bash
npm i notebooklm-client
npx notebooklm export-session
npx notebooklm skill install
```

之后在 Claude Code 里，直接让它通过 `/notecraft chat` 和 NotebookLM 对话即可。

作者给了一段很完整的 Claude prompt 样板，把流程、铁律、输出格式都编码进去。核心就是：

- 先问老师（NotebookLM）
- 老师是只读咨询台
- 带引用原样保留
- 老师答不上就明确说“老师无解”
- Claude 负责执行，不重复喂论文原文

## 4. 实测 token 账

测试设置：

- 语料：47 篇图像 + LiDAR SLAM 相关论文
- 模型：Claude Opus 4.7
- 轮数：5 轮深度问答
- 方式：Claude Code 中正常对话，每轮助手自己调 NotebookLM 问老师

实测结果：5 轮贵档合计 $0.55，平均每轮约 $0.11。

真正决定账单的是 input + cache_creation + output。最关键的点是：47 篇论文原文一个字都没进 Claude 的 cache_creation。

对比传统做法，把 47 篇论文直接塞进 prompt，即使是单 session 多轮对话的最优情况，5 轮问答贵档也会差出 17 倍。

## 5. 学者 / 学生工作流

### 论文 reading list

适合一学期几十篇论文反复查的场景。语料可包括：

- 20-50 篇课题相关论文 PDF
- 课程大纲、讲座字幕
- 导师邮件、章节草稿、读书笔记

可问的问题例如：

- 哪两篇结论互相冲突，冲突在哪一层假设？
- X 方法在这个语料里出现过几次，各自怎么用的？
- A 论文的公式 3 和 B 论文的公式 7 是否等价？

Claude 负责拿 NotebookLM 的答案去写代码、跑实验、整理笔记。

## 6. 打新股 / 读招股书工作流

一本招股书 300-600 页，打新窗口就三天，靠人读完再决策根本来不及。

适合的语料包括：

- 招股书全本
- 基石投资者披露表
- 同行可比公司最新财报
- 保荐人/承销商研报
- 管理层过往访谈、融资估值表
- 行业监管政策文件

可直接问的核心问题包括：

- 核心产品和收入结构
- 与同行差异
- 基石投资者是谁
- 募资用途
- 风险因素哪些是行业共性、哪些是公司特有
- 过往融资估值
- 历史财务是否有一次性收益粉饰
- 关联交易占比

作者强调，NotebookLM 适合把一堆资料压成可引用的决策表，15 分钟扫完就能下单。

## 7. 个人知识库工作流

如果你有一套 Obsidian / Notion / Kindle / Readwise / 工作纪要的全量资料，NotebookLM 可以当一个“第二大脑”。

典型问题：

- 我这三年对“专注力”写过什么？观点变了吗？
- 《原则》和《思考快与慢》对认知偏差的说法哪里重叠哪里冲突？
- 过去一个月会议纪要里，X 项目各人的态度分别是什么？

Claude 负责把 NotebookLM 的多轮答案合成时间轴、观点对比表、待跟进清单。

## 8. 这套方案的边界

作者也讲得很直白：

- 如果语料 < 5k tokens，只查一两次，直接问 Claude 就行
- 如果只是纯 Q&A，不嵌工作流，打开 NotebookLM 网页就够了
- 如果你在乎响应速度超过账单，这套会觉得慢

## 9. 结尾

这套方案的核心就一句话，分工。

- NotebookLM 当老师，答领域知识，带引用，不乱外推
- Claude 当助手，编排工具，写代码，整理结果
- 你当课题负责人，只在关键决策点介入

作者用了一个月，省下来的钱够再吃好几顿大餐。但更重要的是，调研十几篇论文不再让他心疼额度了。
